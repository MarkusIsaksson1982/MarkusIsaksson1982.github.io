Tre nivåer för verkliga projekt

Varje nivå innehåller kort orientering (saker som sällan behandlas djupt i grundkursen i cybersäkerhet), hur din kunskap om hot och risker blir ovärderlig, och hur vi kan samarbeta.

BASIC

Grundläggande testning

4–12 arbetstimmar per uppgift
🧩

Unit Tests + Smoke Tests

Minsta beståndsdelar (enkla funktioner) + snabba “röktester” för att se att systemet lever.

Kort orientering för dig:

Unit tests kollar att en enskild funktion gör exakt det den ska – utan att involvera databas eller nätverk. Smoke tests är som en “quick health check” innan du kör större tester.

Professionellt + säkerhetssynergi:

I ett bank- eller vårdsystem kan unit tests säkerställa att krypteringsfunktioner aldrig läcker data. Du kan bidra med realistiska hotscenarier (t.ex. “tänk om någon skickar malicious input här?”).

Hur vi jobbar:

Du och jag kommer överens om 5–10 kritiska testfall baserat på din kunskap om CIA-triad och vanliga sårbarheter. Sedan skriver och kör jag alla tester (Python pytest eller JavaScript Jest) helt självständigt. Du behöver bara granska resultatet.

“Det här vore spännande att bolla med dig – vilka unit-scenarier skulle du prioritera för en inloggningsmodul?”
INTERMEDIATE

Mellannivå-testning

12–35 arbetstimmar per uppgift
🔗

Integration + Contract + E2E Tests

Databas + tjänst tillsammans • API-kontrakt • Hela användarflöden.

Kort orientering för dig:

Integration testar att olika delar pratar med varandra. Contract Test säkerställer att API:er inte ändras oväntat. E2E simulerar riktiga användare (t.ex. “logga in → betala → få kvitto”).

Professionellt + säkerhetssynergi:

Här blir din kunskap om nätverkssäkerhet, autentisering och compliance guld värd. Vi kan testa “vad händer om en attackerare försöker injecta data mellan tjänsterna?”

Hur vi jobbar:

Du definierar säkerhetskritiska användarflöden och edge-cases. Jag bygger automatiserade tester (Playwright för E2E, pytest + requests för API). När testfallen är godkända kör och underhåller jag allt självständigt – du får regelbundna rapporter.

“Låt oss ta ett exempel från din kurs: hur skulle du vilja testa en E2E-flow för MFA-inloggning?”
ADVANCED

Avancerad testning

25–80+ arbetstimmar per uppgift
🚀

Load + Load & Performance Tests

Simulera många samtidiga användare och extrem belastning.

Kort orientering för dig:

Load tests kollar hur systemet hanterar 1000+ användare samtidigt. Performance tests mäter svarstider och resurser under stress.

Professionellt + säkerhetssynergi:

DDoS-liknande belastning är en verklig hotbild. Du kan bidra med kunskap om vilka tjänster som är mest kritiska ur ett tillgänglighetsperspektiv (Availability i CIA-triad).

Hur vi jobbar:

Du pekar ut de viktigaste “bottlenecks” utifrån din riskkunskap. Jag sätter upp tester med Locust eller k6 (Python/JS). Allt körs lokalt eller i kontrollerade miljöer – jag hanterar exekvering, analys och rapportering självständigt efter att vi enats om scenarierna.

“Vilka belastningsscenarier från verkliga incidenter du lärt dig skulle du vilja simulera här?”
