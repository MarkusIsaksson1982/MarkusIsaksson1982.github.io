Utvalda koncept från kursen & Synergier

Security Ethics & Data Privacy

I "Foundations of Cybersecurity" betonas vikten av etik, lagstiftning (som GDPR) och skydd av personuppgifter (PII/PHI). Att förstå *varför* data måste skyddas är grundläggande.

What can be done with this in professional contexts?

I praktiken innebär detta att man måste bygga arkitektur som bygger på Privacy by Design. När vi integrerar system mot tredjeparter eller moderna AI-modeller måste data hanteras extremt varsamt så att ingen PII läcker ut till externa molnleverantörer.

The CIA Triad & Core Principles

Confidentiality, Integrity, och Availability är hörnstenarna inom Information Security. Modulen täcker hur dessa principer styr en säkerhetsanalytikers vardag.

What can be done with this in professional contexts?

För en säkerhetsanalytiker handlar det om att granska och skapa policys. Men för en utvecklare handlar det om att bygga in *Integrity* via input-validering och *Availability* via robust kodarkitektur.

Synergy Potential: Ett intressant överlapp här är hur man rent praktiskt översätter CIA-triaden till modern webbutveckling. Markus kan bidra med insikter om hur man använder HTML, CSS och JavaScript för att bygga säkra frontend-gränssnitt, samt hur Python kan användas på backend för att säkerställa Data Integrity genom rigorös input-hantering. En diskussion kring hur man bygger system som står emot vanliga injektions-attacker utifrån CIA-ramverket vore mycket givande.

The Threat Landscape: Malware & Social Engineering

Att känna igen attacker såsom phishing, ransomware och olika typer av malware är centralt i första kursmodulen, liksom förståelsen för hur angripare rör sig (Threat Intelligence).

What can be done with this in professional contexts?

Medan en analytiker övervakar SIEM-loggar och utreder incidenter, kan utvecklare bygga automatiserade verktyg som varnar för misstänkt beteende. Att förstå hur social engineering fungerar hjälper också till vid design av UI/UX som "nudgar" användare till att göra säkra val.
