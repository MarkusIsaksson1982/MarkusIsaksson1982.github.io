Autonom Utveckling via Testdriven Design

Inom mjukvaruutveckling finns ett talesätt: "Testen är kontraktet". För dig som fokuserar på cybersäkerhet är det avgörande att system fungerar exakt som förväntat under press. Om vi på förhand kan definiera vilka säkerhetsscenarier som måste passera, kan jag arbeta i stort sett helt oberoende med att bygga logiken och koden för att uppfylla det kontraktet.

Hur vi kan samarbeta asynkront

Konceptet är enkelt: Du tar på dig hatten som Security Analyst / Product Owner och definierar säkerhetskrav. Vi översätter dessa till test-scenarier. Därefter tar jag vid, skriver automatiserade testskript och utvecklar Python eller JS/HTML/CSS-lösningar tills alla tester lyser grönt. Detta minimerar behovet av micromanagement och maximerar vår gemensamma output.

Här är en nedbrytning av hur det kan se ut i praktiken, från grundläggande till avancerade professionella scenarier.

Nivå 1: Basic (Unit Tests & Smoke Tests)
Tidsestimat: 4 till 16 arbetstimmar

Konceptuell Orientering

  • Unit Tests: Testar de absolut minsta byggstenarna i koden (pure functions). Till exempel: Om du skickar in "password123", spottar funktionen ut rätt krypterad hash?
  • Smoke Tests: Ett snabbt yttre test ("ryker det om maskinen när vi slår på den?"). Verifierar att de viktigaste tjänsterna är uppkopplade och svarar på anrop.

Professionellt Scenario: Säker Inloggningsvalidering

Du identifierar att ett internt system behöver en ny funktion för att tvätta (sanitize) användardata så att inga illvilliga tecken kan orsaka en SQL Injection.

Synergy Potential: Du definierar en lista på "onda inputs" (t.ex. ' OR 1=1--). Utifrån detta bygger Markus en svit av Unit Tests i Python. Markus kan därefter helt autonomt skriva koden som hanterar datasaneringen. Arbetet är klart när koden automatiskt stoppar varje hot du definierat, och ett Smoke Test bekräftar att tjänsten fortfarande går att starta.

Nivå 2: Intermediate (Integration & Contract Tests)
Tidsestimat: 16 till 40 arbetstimmar

Konceptuell Orientering

  • Integration Tests: Säkerställer att olika delar av systemet (t.ex. databasen och webbservern) kommunicerar säkert tillsammans.
  • Contract Tests: Kontrollerar att ett API-kontrakt (t.ex. OpenAPI/Swagger) upprätthålls. Om vi lovar att aldrig skicka PII (Personal Identifiable Information) via API:et, säkerställer testet att det aldrig händer.

Professionellt Scenario: Integration mot AI-tjänst med Datasanering

Företaget vill använda en extern AI-modell (cloud-tjänst), men du ser risker med att känslig data kan läcka ut. Vi behöver en "mellanhand" som tvättar datan innan den skickas via API:et.

Synergy Potential: Du och Markus kommer överens om ett API Contract – vilka datafält får skickas och i vilket format. Markus bygger sedan upp arkitekturen (gärna med fokus på local runtimes där möjligt för extra säkerhet) och skriver Integration Tests för att verifiera att hela datakedjan hänger ihop, samtidigt som säkerhetspolicyn bevaras.

Nivå 3: Advanced (E2E, Load & Performance Tests)
Tidsestimat: 40 till 120+ arbetstimmar

Konceptuell Orientering

  • E2E Test (End-to-End): Simulerar en hel användarresa, från att öppna webbläsaren till att klicka på knappar och verifiera att rätt data hamnar säkert i databasen.
  • Load & Performance Tests: Simulerar tung trafik (många concurrent users) för att se hur systemet beter sig under stress.

Professionellt Scenario: Resiliens och Incidentrespons under Belastning

Hur beter sig er nylanserade portal om den utsätts för en massiv trafikvåg (likt en DoS/DDoS-attack)? Fallerar systemet säkert ("fail closed") eller öppnar det upp sårbarheter när minnet tar slut?

Synergy Potential: Här kan vi bygga väldigt intressanta simuleringar. Du specificerar tröskelvärden och incident-parametrar utifrån aktuella hotbilder. Markus skriver komplexa Load Test-skript som belastar systemet artificiellt, samt automatiserade E2E-tester som säkerställer att inloggningsspärrar och larmfunktioner (Alerts) aktiveras precis som de ska när tröskelvärdena passeras. Detta är ett projekt som kräver djup teknisk förståelse för både nätverk och kodstruktur.
